RfK Help Desk

1. {% trans %}Installing the streaming software{% endtrans %}:

In order to participate you need a program which transmits your audio signal to our server. We suggest that you use one of the following clients, but you can look for alternatives here or by browsing the Internet yourself.

Our backend supports the icecast2 streaming protocol, so every client which supports icecast2 should be working fine with RfK.

Tip: We highly recommend edcast when using Windows and idjc when using Linux. Don’t use OSX if you don’t have to. We’re serious.

Windows:

  • edcast (foobar2000 plugin, WinAmp plugin, Standalone)
  • edcast-reborn (WinAmp plugin, Standalone)
  • Mixxx (DJ software with streaming support)

Linux:

  • idjc (Internet DJ Console, experts only)
  • darkice (command-line interface)

Mac OS X:

  • Nicecast (unable to transmit metadata)
  • Ladiocast (unable to transmit metadata)

2. {% trans %}Setting up your streaming software{% endtrans %}:

Connection settings

Here are the settings you need to enter in your streaming client. The exact names will depend on your software

hostname radio.krautchan.net
port 8080
mountpoint /live.ogg
username your RfK-login
password your RfK-password

Encodings

Despite the fact that the mountpoint is called /live.ogg, the server accepts many containers not just Ogg.
Anyhow: we recommend you to use Ogg/Vorbis to be sure since we've encountered some problems when using MP3.
